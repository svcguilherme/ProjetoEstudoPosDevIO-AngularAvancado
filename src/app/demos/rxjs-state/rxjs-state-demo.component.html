🔄 Gerenciamento de Estado com RxJS

BehaviorSubject store, combineLatest e async pipe em um carrinho de compras.

Padrão: BehaviorSubject Store

O BehaviorSubject age como uma "fonte única da verdade" para um slice de estado. Estados derivados são observables criados com operadores RxJS (map, combineLatest, filter…) e consumidos no template com o pipe async. 

// store.ts
private _itens = new BehaviorSubject<ItemCarrinho[]>([]);

readonly itens$ = this._itens.asObservable();           // readonly
readonly total$ = this.itens$.pipe(
  map(itens => itens.reduce((acc, i) => acc + i.preco, 0))
);
readonly resumo$ = combineLatest([this.itens$, this.total$]).pipe(
  map(([itens, total]) => ({ itens, total }))           // combina estados
);

adicionar(p: Produto) {
  // IMUTABILIDADE: sempre crie novos arrays/objetos
  this._itens.next([...this._itens.getValue(), { produto: p, quantidade: 1 }]);
}

// template:
@let resumo = resumo$ | async;  // async pipe: assina e cancela automaticamente
@let vm = resumo$ | async;

RxJS vs Signals — quando usar cada um?

🔄 RxJS (BehaviorSubject)

  • Operações assíncronas complexas (HTTP, WebSocket)
  • Combinação de múltiplas fontes (combineLatest, zip)
  • Transformações com pipeline (switchMap, debounceTime)
  • Estado com tempo / delay / throttle

⚡ Signals

  • Estado de UI local (contador, toggle, form)
  • Valores derivados simples (computed)
  • Integração com Zoneless change detection
  • Mais simples e menos verboso para casos básicos
