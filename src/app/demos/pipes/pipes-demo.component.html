🔧 Pipes — Nativos + Customizados

Transformação de dados nos templates. Pure vs impure. Criação de pipes próprios.

O que é um Pipe?

Um pipe transforma dados no template sem alterar o dado original. Sintaxe: valor | nomePipe:arg1:arg2. Pipes podem ser encadeados: nome | uppercase | slice:0:5. 

// Pure pipe (padrão): Angular só recalcula quando o INPUT muda (por referência)
// ✅ Performance ótima — use sempre que possível.

// Impure pipe (pure: false): recalcula a cada ciclo de change detection
// ⚠️ Cuidado! Pode impactar performance. Use somente quando necessário
//    (ex: pipe que reage a mutações internas de um array sem nova referência).

@Pipe({ name: 'meuPipe', standalone: true, pure: true })
export class MeuPipe implements PipeTransform {
  transform(valor: string, argumento = 'default'): string {
    return // ... lógica de transformação
  }
}

Pipe Customizado: truncar

Trunca texto longo com reticências. Aceita argumentos para limite e sufixo. 

@Pipe({ name: 'truncar', standalone: true })
export class TruncarPipe implements PipeTransform {
  transform(valor: string | null | undefined, limite = 50, sufixo = '...'): string {
    if (!valor) return '';
    if (valor.length <= limite) return valor;
    return valor.substring(0, limite).trimEnd() + sufixo;
  }
}

Pipe Customizado: statusInfo

Retorna um objeto com label, classe CSS e ícone. Use com @let para evitar chamar o pipe múltiplas vezes.

@for (s of statusExemplos; track s) { @let info = s | statusInfo;
"{{ s }}" → {{ info.icone }} {{ info.label }}
}

Pipe Nativo: async

O pipe async assina um Observable ou Promise e exibe o valor emitido. Cancela automaticamente a assinatura quando o componente é destruído — zero memory leak. 

// No componente:
produtos$ = this.produtoService.listar();  // Observable<Produto[]>

// No template:
@let produtos = produtos$ | async;
@if (produtos) {
  @for (p of produtos; track p.id) { ... }
}
💡 Em modo Zoneless, prefira toSignal(obs$) para converter Observables em Signals e eliminar a necessidade do async pipe.
