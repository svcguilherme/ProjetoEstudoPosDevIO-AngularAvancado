🛡️ Tratamento de Erros

Erros síncronos, HTTP e o padrão loading/error/empty/data.

1. Erros síncronos (try/catch)

Use try/catch nos componentes para erros que você espera e sabe tratar. Erros não capturados sobem para o GlobalErrorHandler. 

// Registrar em app.config.ts:
{ provide: ErrorHandler, useClass: GlobalErrorHandler }

// A classe:
export class GlobalErrorHandler implements ErrorHandler {
  handleError(error: unknown) {
    console.error('[Global]', error);
    // monitoringService.log(error); ← Sentry, Datadog...
  }
}

2. Erros HTTP via Interceptor

O ErrorInterceptor (withInterceptors([errorInterceptor])) captura todos os erros HTTP em um único lugar, traduz os códigos de status para mensagens amigáveis e relança como ApiError. 

// shared/interceptors/error.interceptor.ts
export const errorInterceptor: HttpInterceptorFn = (req, next) =>
  next(req).pipe(
    catchError((error: HttpErrorResponse) => {
      const apiError = { status: error.status, mensagem: traduzirErro(error) };
      return throwError(() => apiError);
    })
  );
