🔍 ViewChild & ViewChildren

Acesse elementos do DOM e componentes filhos diretamente no TypeScript.

1. @ViewChild com #templateRef (ElementRef)

Use #nomeRef no template para criar uma referência local. No TypeScript, @ViewChild('nomeRef') captura o ElementRef correspondente. Útil para interagir com o DOM diretamente (foco, leitura de valor, animações). 

// Template:  <input #inputRef />

// TypeScript:
@ViewChild('inputRef') inputEl!: ElementRef<HTMLInputElement>;

ngAfterViewInit() {
  // Só disponível APÓS ngAfterViewInit!
  this.inputEl.nativeElement.focus();
}
@if (!viewInitializado()) {
⚠️ ViewChild só fica disponível após ngAfterViewInit()
}

2. @ViewChild com tipo do componente filho

Passe o tipo da classe do componente filho para capturá-lo. Isso dá acesso a todos os métodos e sinais públicos do filho — o pai pode orquestrar o filho sem usar @Output(). 

@ViewChild(PainelFilhoComponent)
painelFilho!: PainelFilhoComponent;

// Chama método público do filho:
this.painelFilho.destacar();

3. @ViewChildren — lista de componentes

@ViewChildren(Tipo) captura todos os componentes desse tipo no template como um QueryList<T>. O QueryList emite changes quando itens são adicionados ou removidos dinamicamente. 

@ViewChildren(PainelFilhoComponent)
todosPaineis!: QueryList<PainelFilhoComponent>;

chamarTodos() {
  this.todosPaineis.forEach(p => p.destacar());
}
@for (n of itensPaineis; track n) { }
