⚡ NgZone — Controle de Change Detection

runOutsideAngular, zone.run() e o impacto no ciclo de detecção de mudanças.

O que é a NgZone?

A NgZone é uma wrapper sobre a library Zone.js. Ela "intercepta" operações assíncronas (setTimeout, events, fetch…) e, ao terminar cada uma, avisa o Angular para verificar mudanças na UI.

📌 Este projeto usa Zoneless Change Detection (provideZonelessChangeDetection()). A NgZone ainda está disponível para compatibilidade com libs de terceiros, mas o Angular não depende mais dela para detectar mudanças — usa Signals diretamente. 
// Zone-based (padrão até Angular 17):
//   Angular usa Zone.js para saber quando verificar a UI.
//   Cada setTimeout, click, fetch → Zone notifica → Angular roda CD.

// Zoneless (Angular 18+, este projeto):
//   Angular usa Signals para saber o que mudou.
//   Zone.js pode ser removido do bundle (economia > 30KB).
//   provideZonelessChangeDetection() → sem Zone.js obrigatório.

1. runOutsideAngular() — código sem CD

Use para tarefas que rodam frequentemente e não precisam atualizar a UI imediatamente (animações, polling de sensores, workers). Em apps Zone-based, reduz o número de ciclos de CD. Em Zoneless, o comportamento é o mesmo — Signals notificam quando mudam. 

constructor() {
  const zone = inject(NgZone);

  zone.runOutsideAngular(() => {
    setInterval(() => {
      // não dispara CD em apps Zone-based
      // Signal ainda reage mesmo aqui em apps Zoneless
      this.contador.update(v => v + 1);
    }, 300);
  });
}

2. zone.run() — re-entrar na Zone

Libs de terceiros (Google Maps, Socket.io, Chart.js) disparam callbacks fora da Zone. Sem zone.run(), em apps Zone-based, a UI não atualiza. Envolva a atualização em zone.run() para forçar CD. 

// Simula callback de lib externa (roda fora da Zone Angular)
this.zone.runOutsideAngular(() => {
  socket.on('mensagem', (dados) => {
    // ⚠️ Zone-based: UI NÃO atualiza aqui fora
    this.zone.run(() => {
      // ✅ Força re-entrada na Zone → CD dispara → UI atualiza
      this.mensagem.set(dados);
    });
  });
});
