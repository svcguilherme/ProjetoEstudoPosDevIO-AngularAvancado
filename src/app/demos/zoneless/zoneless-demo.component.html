Angular v19+

Zoneless Change Detection

Detecção de mudanças sem Zone.js — o Angular atualiza a view somente quando um Signal muda, com precisão cirúrgica.

⚙️ Como funciona?

🔴 Modo Clássico (Zone.js)

  1. Zone.js patcha APIs globais: setTimeout, Promise, fetch…
  2. Toda operação assíncrona aciona change detection em toda a árvore de componentes.
  3. Angular verifica cada binding — mesmo os que não mudaram.
  4. ✅ Fácil de usar,
    ❌ pode ser ineficiente em apps grandes.

🟢 Modo Zoneless (Signals)

  1. Sem Zone.js — nenhuma API global é patcheada.
  2. O Angular rastreia quais Signals são lidos por cada trecho do template.
  3. Quando um signal muda, apenas os nós que o leram são re-renderizados.
  4. ✅ Performance máxima,
    ✅ previsibilidade total.

🚀 Como habilitar Zoneless no seu projeto

1
app.config.ts — adicione provideZonelessChangeDetection(): 
import { provideZonelessChangeDetection } from '@angular/core';

export const appConfig: ApplicationConfig = {
  providers: [
    provideZonelessChangeDetection(), // ← aqui
    provideRouter(routes),
  ]
};
2
angular.json — remova o polyfill do zone.js (opcional, mas recomendado para reduzir bundle): 
"polyfills": []  // remova "zone.js"
3
Substitua estado mutable por Signals em seus componentes. Observables continuam funcionando via async pipe (que chama markForCheck() internamente).

📋 Cheat Sheet — Signals API

signal(valor) Cria um signal com valor inicial
signal.set(x) Substitui o valor
signal.update(fn) Atualiza com base no valor atual
computed(() => ...) Valor derivado, memoizado
effect(() => ...) Side-effect reativo
toSignal(obs$) Converte Observable → Signal
toObservable(sig) Converte Signal → Observable
linkedSignal() Signal com fonte dinâmica (v19+)
