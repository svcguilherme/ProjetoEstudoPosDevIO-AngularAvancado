🔍 Filtros Reativos com Signals

computed() para filtrar listas — sem impure pipes, sem performance issues.

⚠️ Por que não usar Impure Pipe para filtros?

// ❌ Impure pipe (pure: false) — EVITE para filtros:
//    recalcula a CADA ciclo de change detection (centenas de vezes/segundo)
//    Em listas grandes → lag visível.

@Pipe({ name: 'filtrar', pure: false })  // ← puro: false
export class FiltrarPipe {
  transform(lista: Produto[], filtro: string) {
    return lista.filter(...);  // chamado em TODA checagem de CD
  }
}

// ✅ Correto: computed() recalcula APENAS quando o signal de filtro muda
produtosFiltrados = computed(() =>
  PRODUTOS.filter(p => p.nome.toLowerCase().includes(this.termoBusca()))
);

Como funciona

// Signals independentes para cada filtro
termoBusca      = signal('');
categoriaFiltro = signal('Todas');
statusFiltro    = signal('todos');

// computed lê os 3 signals automaticamente
// Angular rastreia as dependências e recalcula quando QUALQUER uma muda
produtosFiltrados = computed(() => {
  const termo     = this.termoBusca().toLowerCase();
  const categoria = this.categoriaFiltro();
  const status    = this.statusFiltro();

  return PRODUTOS.filter(p =>
    p.nome.toLowerCase().includes(termo) &&
    (categoria === 'Todas' || p.categoria === categoria) &&
    (status === 'todos' || p.status === status)
  );
});
